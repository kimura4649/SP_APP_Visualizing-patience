# 我慢貯金アプリ 仕様書

## コンセプト
日常の「我慢」を金額として記録・可視化するアプリ。
シンプル・軽量・継続しやすさ最優先。

- **プラス金額**：我慢できた金額（例：コンビニを我慢して+¥500）
- **マイナス金額**：誘惑に負けて浪費してしまった金額（例：外食してしまった -¥800）
- 合計金額がマイナスの場合も `¥-XXX` 形式でそのまま表示する

## 画面構成

### 画面①：メイン画面

**画面エリア構成（縦比率）**

| エリア | 縦比率 | 内容 |
|--------|--------|------|
| 上部エリア | 40% | 日付・合計金額 |
| 中央エリア | 30% | ボタン群 |
| 下部エリア | 30% | 広告 |

**レイアウト（上から順）**

1. ステータスバー（時刻 / バッテリーなど）
2. ヘッダー
   - 左：タイトル「我慢貯金」
   - 右：「履歴 →」ボタン（タップで画面②へ）
3. 上部エリア（画面縦40%）
   - 右上：カレンダーアイコン（タップでカレンダーモーダルを表示）
   - `{月}月の我慢貯金` ラベル
   - 合計金額（大きく表示、¥XXX形式）
   - 区切り線
   - 「今日の合計」と当日合計金額（端末のローカルタイムゾーンに基づき当日0:00:00〜23:59:59の記録を集計、0:00にリセット）
     - データなし時：「{MM}月{DD}日（システム日付）：¥0」と表示
   - `{前月}月の実績` ラベル ＋ 合計金額（サブトーン）
4. 中央エリア（画面縦30%）
   - 登録済みボタン群（3列グリッド）※登録があるときのみ表示
     - **デフォルトボタン群より上に表示**（ユーザーが登録したボタンを優先表示）
     - 各ボタンは金額＋ボタン名の2行構成（デフォルトボタンと同形式）
     - ボタン削除時：残ったボタンが左詰めで3列グリッドに並び直す（空きマスを作らない）
   - デフォルトボタン群（3列グリッド）
     - 各ボタンに金額＋カテゴリ名の2行構成
     - 初期出荷時の設定値（変更・削除可能）：
       - +¥150「ジュース」
       - +¥500「コンビニ」
       - +¥800「外食」
       - +¥300「お菓子」
       - +¥1,200「ランチ」
       - +¥2,000「飲み会」
     - ボタン削除時：残ったボタンが左詰めで3列グリッドに並び直す（空きマスを作らない）
   - 登録済みボタン・デフォルトボタンともに長押し（Androidの標準アプリ長押しと同等の時間）で削除ボタンを表示、タップで削除。削除ボタン以外の領域をタップまたは3秒経過で削除ボタンを非表示に戻す
   - カスタム入力（カード形式でまとめる）
     - 金額入力フィールド ＋「追加」ボタン
     - メモ入力フィールド（任意）
     - チェックボックス「ワンタップボタンに登録する」
       - チェックON時：ボタン名入力フィールドを展開表示
       - 追加ボタン押下時に記録への追加と同時にボタンを登録
       - チェックOFF時：記録のみ追加（従来動作）
5. 下部エリア（画面縦30%）
   - 広告バナー（320×50、読み込み失敗時は空白）

### モーダル：カレンダー

カレンダーアイコンタップで表示されるモーダル。

**レイアウト**

1. ヘッダー
   - 左：「‹」前月ボタン（最初のレコードが存在する月では非活性）
   - 中央：`{YYYY}年{M}月` 表示
   - 右：「›」翌月ボタン（現在月では非活性）
2. 月合計
   - `{M}月 合計：¥XXX` 表示（データなし時は¥0）
3. カレンダーグリッド（7列×最大6行）
   - 各日付セルに当日の合計金額を小さく表示
   - データなし日は金額非表示（空白）
   - 当日セルは背景色でハイライト表示（色はデザイン時に決定）
4. 閉じるボタン（右上）

---

### 画面②：履歴画面

**レイアウト（上から順）**

1. ヘッダー
   - 左：「← もどる」ボタン（メイン画面へ戻る）
   - 中央：`{月}月` タイトル
2. 月切り替えコントロール
   - 「‹ 前月」ボタン ／ 「YYYY年M月」表示 ／ 「翌月 ›」ボタン
3. 月合計バー
   - 「合計」ラベル ＋ ¥XXX金額
4. 履歴リスト
   - 日付ごとにグループ化、日付は昇順（古い順）で表示
   - 日付ヘッダー形式：`M月D日（曜日）`（例：4月9日（木））
   - 各レコード：メモ（空の場合は金額のみ表示） ＋ ¥XXX
   - 同一日内のレコードは登録順（古い順）で表示
5. 広告バナー（下部固定、320×50、読み込み失敗時は空白）

## インタラクション

| 操作 | 結果 |
|---|---|
| ワンタップボタンを押す | 即座に金額追加。「+¥XXX（カテゴリ名）を追加しました」トースト表示（2秒で消える） |
| カスタム金額を入力して「追加」 | 任意金額＋メモを追加、フィールドをクリア |
| 0円で「追加」を押す | エラートースト「0円は記録できません」を表示（2秒で消える）、登録しない |
| 「ワンタップボタンに登録する」にチェック | ボタン名入力欄が展開される（ボタン名は空でも登録可） |
| チェックONで「追加」を押す | 記録追加と同時にボタンを登録。「+¥XXX を追加 ＆「○○」を登録しました」トースト表示（2秒で消える） |
| ボタンを長押し | 削除ボタンを表示（デフォルト・登録済み共通） |
| 削除ボタンをタップ | ボタンを削除 |
| 「履歴 →」を押す | 履歴画面へ遷移 |
| 「← もどる」を押す | メイン画面へ戻る |
| 月切り替えボタンを押す | 表示月が変わり、対応する記録を表示 |
| 前月ボタンを押す（最初のレコードが存在する月の場合） | 非活性化（利用開始前には遡れない） |
| カレンダーアイコンをタップ | カレンダーモーダルを表示 |
| カレンダーモーダルの前月・翌月ボタン | 表示月を切り替え（翌月は現在月、前月は利用開始月で非活性） |
| カレンダーモーダルの閉じるボタン | モーダルを閉じる |
| 追加成功時 | 「+¥XXX を追加しました」トースト（2秒で消える） |

## データ構造

### レコード単位

```
{
  id: string,      // 3桁の連番ID（例："001"）。追加時に現在の最大IDに+1して採番
  date: "YYYY-MM-DD",
  amount: number,  // 円（-100,000 〜 100,000）
  memo: string     // 任意、空文字可（最大500文字）
}
```

### 退避レコード単位

削除されたレコードを保持する領域。キー：`archivedRecords`

```
{
  id: string,      // 削除時点のID
  date: "YYYY-MM-DD",
  amount: number,
  memo: string,
  deletedAt: "YYYY-MM-DD" // 削除日
}
```

**ID管理ルール**
- レコード追加時：現在の `records` 内の最大IDに+1して採番（3桁ゼロ埋め、例："001"〜"999"）
- レコード削除時：該当レコードを `archivedRecords` に移動 → `records` 全体のIDを先頭から連番で振り直す

### 登録済みボタン単位

```
{
  amount: number,  // 円（-100,000 〜 100,000）
  name: string     // ボタン名（例：「タクシー」、最大30文字）
}
```

### バリデーション仕様

| 項目 | ルール |
|------|--------|
| 金額入力範囲 | -100,000円 〜 100,000円（0円不可、小数不可） |
| 0円入力時 | エラートースト表示、登録不可 |
| メモ文字数上限 | 500文字 |
| ボタン名文字数上限 | 30文字（空文字可） |
| 登録済みボタン上限 | 30個（上限到達時は登録ボタンを非活性化） |
| 翌月ボタン | 現在月では非活性化（未来の月へは遷移不可） |
| 前月ボタン | 最初のレコードが存在する月では非活性化 |
| 履歴遡り範囲 | 最初のレコードが存在する月まで（全データを保持） |

### アーキテクチャ設定
- `newArchEnabled: false`（旧アーキテクチャで動作）

### データ保存方針
- サーバー不要、端末内にローカル保存
- 記録データ・登録済みボタンともにAsyncStorageで永続化
- キー設計例：`records`、`registeredButtons`
- オフライン動作：記録の追加・閲覧はオフラインで完全動作。広告はオフライン時は非表示（空白）
- AsyncStorage書き込み失敗時：エラートースト「保存に失敗しました。再度お試しください」を表示（2秒で消える）

### AsyncStorage容量管理

AsyncStorageの最大容量を6MBとして管理する。容量チェックはアプリ起動時および記録追加後に実行。

| 状態 | 条件 | 動作 |
|------|------|------|
| 警告 | 使用量が最大容量の80%以上 | 警告ダイアログを表示し、メール移行を促す |
| 強制削除 | 使用量が最大容量の90%以上 | 古いデータを自動削除（削除後に通知） |

**警告ダイアログ仕様（80%時）**
- メッセージ：「データ容量が上限に近づいています。古いデータをメールで書き出すことができます。」
- 入力項目：送信先メールアドレス・移行開始日（FROM）・移行終了日（TO）
- ボタン：「メールで送信して削除」「あとで」
- 「メールで送信して削除」押下時：指定期間のデータをCSV形式でメール送信（expo-mail-composerで端末のデフォルトメールアプリを起動・CSV添付）後、該当データを削除
- 「あとで」選択後：90%到達時に再度ダイアログを表示（それまでは非表示）

**CSV仕様**
- カラム：`date,amount,memo`
- 文字コード：UTF-8（BOM付き）
- 区切り文字：カンマ
- 改行：CRLF

**強制削除仕様（90%時）**
- メールアドレスが設定済みの場合：全データをCSVでメール送信してから最も古い月のデータを月単位で削除し、使用量が80%未満になるまで繰り返す
- メールアドレスが未設定の場合：最も古い月のデータを月単位で削除し、使用量が80%未満になるまで繰り返す
- 削除完了後にトースト通知：「容量確保のため古いデータを削除しました」（2秒で消える）

- メイン画面への直近記録プレビュー
