# 我慢貯金アプリ 仕様書

## コンセプト
日常の「我慢」を金額として記録・可視化するアプリ。
シンプル・軽量・継続しやすさ最優先。

- **プラス金額**：我慢できた金額（例：コンビニを我慢して+¥500）
- **マイナス金額**：誘惑に負けて浪費してしまった金額（例：外食してしまった -¥800）
- 合計金額がマイナスの場合も `¥-XXX` 形式でそのまま表示する

## 画面構成

### 画面①：メイン画面

**レイアウト（上から順）**

1. ステータスバー（時刻 / バッテリーなど）
2. ヘッダー
   - 左：タイトル「我慢貯金」
   - 右：「履歴 →」ボタン（タップで画面②へ）
3. ヒーローカード
   - `{月}月の我慢貯金` ラベル
   - 合計金額（大きく表示、¥XXX形式）
   - 区切り線
   - 「今日の合計」と当日合計金額
4. 前月実績バー
   - `{前月}月の実績` ラベル ＋ 合計金額（サブトーン）
5. ワンタップ追加ボタン群（3列グリッド）
   - 各ボタンに金額＋カテゴリ名の2行構成
   - 現在の初期値：
     - +¥150「ジュース」
     - +¥500「コンビニ」
     - +¥800「外食」
     - +¥300「お菓子」
     - +¥1,200「ランチ」
     - +¥2,000「飲み会」
6. カスタム入力（カード形式でまとめる）
   - 金額入力フィールド ＋「追加」ボタン
   - メモ入力フィールド（任意）
   - チェックボックス「ワンタップボタンに登録する」
     - チェックON時：ボタン名入力フィールドを展開表示
     - 追加ボタン押下時に記録への追加と同時にボタンを登録
     - チェックOFF時：記録のみ追加（従来動作）
7. 登録済みボタン群（3列グリッド）※登録があるときのみ表示
   - デフォルトボタン群の下に「登録済みボタン」セクションとして表示
   - 各ボタンは金額＋ボタン名の2行構成（デフォルトボタンと同形式）
8. 広告バナー（下部固定、320×50）

---

### 画面②：履歴画面

**レイアウト（上から順）**

1. ヘッダー
   - 左：「← もどる」ボタン（メイン画面へ戻る）
   - 中央：`{月}月` タイトル
2. 月切り替えコントロール
   - 「‹ 前月」ボタン ／ 「YYYY年M月」表示 ／ 「翌月 ›」ボタン
3. 月合計バー
   - 「合計」ラベル ＋ ¥XXX金額
4. 履歴リスト
   - 日付ごとにグループ化（例：「4月1日（水）」）
   - 各レコード：メモ ＋ +¥XXX
5. 広告バナー（下部固定）

## インタラクション

| 操作 | 結果 |
|---|---|
| ワンタップボタンを押す | 即座に金額追加、トースト通知表示 |
| カスタム金額を入力して「追加」 | 任意金額＋メモを追加、フィールドをクリア |
| 「ワンタップボタンに登録する」にチェック | ボタン名入力欄が展開される |
| チェックONで「追加」を押す | 記録追加と同時にボタンを登録。「+¥XXX を追加 ＆「○○」を登録しました」トースト表示 |
| 「履歴 →」を押す | 履歴画面へ遷移 |
| 「← もどる」を押す | メイン画面へ戻る |
| 月切り替えボタンを押す | 表示月が変わり、対応する記録を表示 |
| 追加成功時 | 「+¥XXX を追加しました」トースト（2秒で消える） |

## データ構造

### レコード単位

```
{
  date: "YYYY-MM-DD",
  amount: number,  // 円（-100,000 〜 100,000）
  memo: string     // 任意、空文字可（最大500文字）
}
```

### 登録済みボタン単位

```
{
  amount: number,  // 円（-100,000 〜 100,000）
  name: string     // ボタン名（例：「タクシー」、最大30文字）
}
```

### バリデーション仕様

| 項目 | ルール |
|------|--------|
| 金額入力範囲 | -100,000円 〜 100,000円（0円不可、小数不可） |
| メモ文字数上限 | 500文字 |
| ボタン名文字数上限 | 30文字（必須） |
| 登録済みボタン上限 | 30個（上限到達時は登録ボタンを非活性化） |
| 翌月ボタン | 現在月では非活性化（未来の月へは遷移不可） |

## システム仕様

### 対応プラットフォーム
- iOS / Android（クロスプラットフォーム）

### 技術スタック

| 役割 | 採用技術 |
|---|---|
| フレームワーク | React Native + Expo |
| 言語 | JavaScript（TypeScriptは後回しでOK） |
| データ保存 | AsyncStorage（端末内ローカル保存） |
| 広告 | expo-ads-admob |

### データ保存方針
- サーバー不要、端末内にローカル保存
- 記録データ・登録済みボタンともにAsyncStorageで永続化
- キー設計例：`records`、`registeredButtons`

### 開発環境
- Expo Go アプリで実機確認可能（Android / iOS 両対応）
- Node.js + Expo CLI
